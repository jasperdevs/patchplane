# Brand Strategy

Skill used: `lunour-branding-playbook`

## Positioning

For open-source maintainers and AI-heavy product teams who need a reliable home for code, Patchplane is a modern software forge that makes review, CI, releases, and agent context feel like one fast workflow because it is built around source history and intent from day one.

## Category

Modern software forge.

Avoid calling the product only "Git hosting." That makes the product sound like commodity infrastructure. Patchplane is the work surface around source code.

## Competitive Alternatives

- Stay on GitHub and tolerate reliability/product trust concerns.
- Move to GitLab for enterprise completeness.
- Move to Bitbucket for Atlassian integration.
- Move to Forgejo/Codeberg for open governance.
- Add Graphite on top of GitHub for better review.
- Wait for agent-native infrastructure like Pierre or Entyre.

## Unique Attributes To Build Toward

1. Taste-led hosted forge with a monochrome, shadcn-compatible product system.
2. Review, CI, releases, and agent context as one timeline.
3. Open-core and portable by default.
4. Push-to-create and low-ceremony CLI flows.
5. Profile and contribution portability to reduce GitHub network loss.

## Value

- Maintainers get a calmer way to review and ship.
- Teams get reliable code hosting without surrendering portability.
- Agent-heavy workflows get durable context instead of scattered session logs.
- Open-source projects get continuity, not just migration tooling.

## Messaging Hierarchy

One-liner:

> Source control that feels as fast as deploy.

Elevator pitch:

Patchplane is a modern software forge for teams that want GitHub's community value without GitHub's ceremony and trust risk. It brings repository hosting, review, CI, releases, and agent context into one clean workflow. The long-term goal is open-core infrastructure that developers can trust, host, and leave without losing their history.

Core messages:

- Faster source workflows: repo creation, review, CI, and releases should be one path.
- Context survives the session: agent and human decisions stay attached to changes.
- Open by design: exports, mirrors, and self-hosting are part of the trust model.
- Community matters: profiles and contribution proof must survive migration.
- Taste is infrastructure: high-quality UI reduces mistakes and increases trust.

## Voice

Direct, calm, precise, and technical.

Is:

- Clear about tradeoffs.
- Comfortable saying no.
- Specific about product behavior.
- Quietly premium.

Is not:

- Corporate DevSecOps soup.
- Hype-first AI language.
- Cutesy open-source nostalgia.
- Generic "developer productivity platform" copy.

## Visual Direction

Monochrome first. Use black, graphite, white, and restrained green/blue accents only when state needs meaning. The product should feel like shadcn, Vercel, and a serious terminal tool rather than a marketing dashboard.

Typography:

- Inter for interface and marketing text.
- JetBrains Mono for commit hashes, commands, timings, diffs, versions, and tabular metrics.

Motion:

- Follow Fluid Functionalism: motion explains state, hover, grouping, and causality.
- Use spring-like response for controls, but never autoplay decorative motion.
- Respect reduced motion.
