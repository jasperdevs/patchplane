# Naming

Skill used: `lunour-naming`  
Working recommendation: **Patchplane**

## Inferred Discovery

- Company: a hosted and eventually open-source software forge for Git hosting, review, CI, releases, and agent context.
- Primary customer: open-source maintainers, AI-heavy product teams, and small dev teams that care about speed, trust, and taste.
- Desired feeling: calm control, momentum, trust, and technical sharpness.
- Personality: precise, minimal, reliable, opinionated.
- Loved references: Vercel, shadcn, monochrome systems, Fluid Functionalism, polished developer tools.
- Constraints: Inter and JetBrains Mono, monochrome design, shadcn-compatible theme, open-source direction, avoid cloning competitors.

## Recommendation

Use **Patchplane** as the working name.

Brand story: Patches are where code changes become shared work. Patchplane is the clean plane where humans and agents review, test, release, and understand those changes together.

Tagline options:

- Source control that feels as fast as deploy.
- Review, CI, and agent context on one plane.
- A calmer forge for shipping code.

Domain strategy:

- Check `patchplane.com` first.
- Fallbacks: `patchplane.dev`, `getpatchplane.com`, `patchplane.app`, `patchplane.sh`.
- Handles: `patchplane`, `patchplanehq`, `getpatchplane`.

Required before public launch:

- USPTO search for PATCHPLANE and phonetic variants.
- WIPO search if global use is planned.
- Domain and social handle check.
- Legal clearance before investing in a paid domain or public trademark usage.
